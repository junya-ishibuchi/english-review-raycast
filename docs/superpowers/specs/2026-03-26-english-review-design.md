# English Review Tool — Design Spec

## Overview

Raycastでの英語学習やり取りを自動記録し、弱点を解析・可視化して復習できるパーソナルツール。

## 背景・課題

- AIに英文訂正・翻訳を日常的に依頼しているが、その結果が全てチャット履歴に埋もれている
- 訂正・翻訳のやり取りは本人の英語力の弱点そのものであり、復習すれば効率的にスキルアップできる
- 記録の手間が大きいと続かないため、既存ワークフローに自然に組み込む必要がある

## ユーザー

- 個人利用（1人）
- 1日10件以上の記録が発生する
- 主な作業環境: macOS + Raycast

## Raycastコマンド

4つのコマンドで全ユースケースをカバーする。

| コマンド | ホットキー (仮) | 入力 | 温度感 | 出力 |
|---|---|---|---|---|
| **Translate** | `⌥+T` | 日本語 | 書けない → 教えて | 英訳 |
| **Correct** | `⌥+C` | 自分の英文 | 合ってるかわからない → 直して | 訂正後の英文 + 変更箇所ハイライト |
| **Explain** | `⌥+E` | 英語表現 | 読めない → 教えて | 意味・用法・例文 |
| **Confirm** | `⌥+F` | 英語表現 | たぶん読めてるけど → 確認して | 確認結果 |

### 共通フロー

1. ホットキーでコマンド起動
2. テキスト入力 → Enter
3. AIの結果が即座に表示される
4. 結果はクリップボードに自動コピー
5. バックグラウンドでJSON記録 + 弱点解析（ユーザーは待たない）

### 複数往復の会話

- Raycast AIチャットで会話を続ける
- 各コマンドの初回応答は自動記録される
- 深掘りして追加のやり取りをした場合、最終的な結論・訂正結果を記録として採用する
- 実装の詳細はRaycast AIチャットのAPI仕様を確認した上で決定する

## データモデル

### 記録 (Record)

```json
{
  "id": "uuid",
  "type": "correction | translation | explanation | confirmation",
  "input": "ユーザーの入力テキスト",
  "output": "AIの結果テキスト",
  "analysis": {
    "reason": "なぜ間違えたか / なぜこの表現か",
    "key_point": "学習のツボ（ハイライトすべき箇所）",
    "categories": ["冠詞", "時制"],
    "difficulty": "basic | intermediate | advanced"
  },
  "created_at": "2026-03-26T10:30:00Z",
  "review_count": 0
}
```

### カテゴリ体系（初期版、AI自動分類）

**文法**
- 冠詞 (a/the/無冠詞)
- 時制 (現在完了 vs 過去形 等)
- 前置詞
- 単数/複数
- 関係代名詞
- 助動詞
- 受動態/能動態

**表現**
- 直訳的（日本語の構造をそのまま英語にしている）
- フォーマル/カジュアルの不一致
- 不自然な語順
- 冗長な表現

**語彙**
- 類義語の選択ミス
- コロケーション（単語の組み合わせ）
- イディオム

カテゴリは固定ではなく、AIが新しいパターンを検出した場合は自動追加される。

## データストレージ

### JSONファイル構成

```
iCloud Drive/EnglishReview/
├── data/
│   ├── 2026-03.json        ← 月別にデータ蓄積
│   ├── 2026-04.json
│   └── categories.json     ← カテゴリマスタ
├── daily/
│   ├── 2026-03-26.html     ← 日次復習HTML
│   └── ...
├── weekly/
│   ├── 2026-W13.html       ← 週間サマリーHTML
│   └── ...
└── assets/
    └── style.css           ← 共通スタイル
```

- 月別JSONで分割し、ファイルサイズを小さく保つ
- iCloud Drive に配置し、PC/iPhone間で自動同期
- SQLiteではなくJSONを選択した理由: iCloud同期との相性が良い、テキストエディタで中身を確認できる、年間3,650件程度なのでパフォーマンス問題なし

## 復習HTML

### 日次HTML（毎日生成）

前日分の記録をリッチなHTMLで表示:

- 訂正: 訂正前後のdiff表示（赤/緑ハイライト）
- 翻訳: 日本語 → 英訳 + 解説
- 解説: 英語表現の意味・用法
- 確認: 確認結果
- 各記録に弱点カテゴリタグを表示
- なぜ間違えたか、なぜこの表現かの解説をハイライト表示

### 週次HTML（毎週生成）

- 弱点カテゴリの出現回数ランキング
- 先週比での改善/悪化の推移
- 最も繰り返している間違いパターンTOP5
- 今週のハイライト（よくできた表現など）

### 生成の仕組み

- macOS launchd (cron相当) で毎朝実行
- JSONデータを読み込み → HTMLテンプレートに流し込み → iCloudフォルダに出力
- JavaScriptも使えるため、フィルタリングやインタラクティブ表示も可能

## 技術スタック

| レイヤー | 技術 | 理由 |
|---|---|---|
| 入力 | Raycast Extension (TypeScript) | 既存ワークフローに統合 |
| AI | Claude API (Anthropic) | 翻訳/訂正/解説 + 弱点解析 |
| データ | JSONファイル (iCloud Drive) | シンプル、iCloud同期と相性良好 |
| 復習UI | 静的HTMLファイル (iCloud Drive) | ブラウザで閲覧、PC/iPhone対応 |
| スケジュール | macOS launchd | HTML生成の定期実行 |

## 将来の拡張

- **Webダッシュボード**: JSON→DB移行 + APIサーバーデプロイ。Raycastのリクエスト先URLを変更するだけで移行可能
- **弱点パターンの可視化強化**: インタラクティブなグラフ、ドリルダウン分析
- **同じ間違いの分類・可視化**: 苦手分野の傾向を長期的に追跡

## AI解析プロンプト

プロンプトは実際のアウトプットを見ながら繰り返し調整する前提。初期版では:

- コマンドタイプに応じた解析を実行
- 既存カテゴリに当てはまらなければ新しいカテゴリを提案
- 解説は「なぜ」にフォーカス（なぜ間違いか、なぜこの表現が自然か）
