# English Review Tool — Design Spec

## Overview

Raycastでの英語学習やり取りを自動記録し、弱点を解析・可視化して復習できるパーソナルツール。

## 背景・課題

- AIに英文訂正・翻訳を日常的に依頼しているが、その結果が全てチャット履歴に埋もれている
- 訂正・翻訳のやり取りは本人の英語力の弱点そのものであり、復習すれば効率的にスキルアップできる
- 記録の手間が大きいと続かないため、既存ワークフローに自然に組み込む必要がある

## ユーザー

- 個人利用（1人）
- 1日10件以上の記録が発生する
- 主な作業環境: macOS + Raycast

## Raycastコマンド

3つのコマンドで全ユースケースをカバーする。

| コマンド | 入力 | 用途 | 出力 |
|---|---|---|---|
| **English** | 日本語 | 英語にできない → 翻訳して | 英訳 |
| **Correct** | 自分の英文 | 合ってるかわからない → 直して | 訂正後の英文 + 日本語訳 |
| **Nihongo** | 英語 | 読めない → 日本語にして | 日本語訳 |

### 共通フロー

1. テキストを選択した状態でホットキーを押す → 選択テキストを自動取得して即実行
2. テキスト未選択の場合 → フォーム入力にフォールバック
3. AIの結果が即座に表示される
4. **Enter** → 結果が元のアプリにペーストされ（選択部分を置換）、Raycastが閉じる
5. バックグラウンドでJSON記録 + 弱点解析（ユーザーは待たない）

### Follow Up（会話の継続）

- 結果画面で「Follow Up」アクションを選択すると、追加の指示を入力できる
- 元の入力 + 前回の結果 + フォローアップ指示をAIに送り、結果を更新
- 例: 「もっとカジュアルに」「ビジネス寄りのニュアンスで」
- Follow Upは何度でも繰り返し可能
- Follow Up後の結果も自動記録される

### Correct固有の機能

- 訂正結果（英語）を表示した後、バックグラウンドで日本語訳を取得して追加表示
- ペースト対象は英語の訂正結果のみ（日本語訳はペースト対象外）

## AIモデル構成

2つのモデルを使い分ける:

| 用途 | モデル | 理由 |
|---|---|---|
| 結果生成（翻訳・訂正） | Google Gemini 3.1 Flash Lite | 高速。ユーザーの待ち時間を最小化 |
| 弱点解析 | Anthropic Claude 4.6 Sonnet | 高品質。解析の深さ・正確性を重視 |

- Raycast組み込みAIを使用（外部APIキー不要、Raycast Proサブスクリプションに含まれる）
- 追加課金なし

### AI解析プロンプト

タイプごとに解析の観点が異なる:

- **Correction**: 文法・表現の何が間違っていたか、一文ごとに具体的に解説
- **Translation (JP→EN)**: なぜこの英語表現になるか、日本語との構造の違い
- **ToJapanese (EN→JP)**: なぜこの英語が読みにくかったか、構文・語彙・文法の解説

共通ルール:
- reason, key_pointは日本語で出力
- 一文ごとに分割して解析
- カテゴリは既存ラベルに当てはまらなければ新規作成

### Correctionプロンプトの方針

- 元の文体・構成をできるだけ維持し、文法的に間違っている箇所や明らかに不自然な箇所だけ直す
- 文体・構造そのものが不自然な場合は構成を変えてもよい
- カジュアルな表現は意図的なものとして尊重し、無理にフォーマルに直さない

## データモデル

### 記録 (Record)

```json
{
  "id": "uuid",
  "type": "correction | translation | toJapanese",
  "input": "ユーザーの入力テキスト",
  "output": "AIの結果テキスト",
  "analysis": {
    "sentences": [
      {
        "original": "元の一文",
        "corrected": "訂正/翻訳後の一文",
        "reason": "なぜこの訂正/翻訳が必要だったか（日本語）",
        "key_point": "学習ポイント（日本語）"
      }
    ],
    "categories": ["冠詞", "時制"],
    "difficulty": "basic | intermediate | advanced"
  },
  "created_at": "2026-03-26T10:30:00Z",
  "review_count": 0
}
```

### カテゴリ体系（AI自動分類）

**文法**: 冠詞, 時制, 前置詞, 単数/複数, 関係代名詞, 助動詞, 受動態/能動態

**表現**: 直訳的, フォーマル/カジュアル, 語順, 冗長な表現, イディオム

**語彙**: 類義語, コロケーション

カテゴリは固定ではなく、AIが新しいパターンを検出した場合は自動追加される。

## データストレージ

### JSONファイル構成

```
iCloud Drive/EnglishReview/
├── data/
│   ├── 2026-03.json        ← 月別にデータ蓄積
│   └── categories.json     ← カテゴリ集計
├── daily/
│   └── 2026-03-27.html     ← 日次復習HTML
├── weekly/
│   └── 2026-03-27.html     ← 週間サマリーHTML
└── assets/
    └── style.css           ← 共通スタイル
```

- 月別JSONで分割し、ファイルサイズを小さく保つ
- iCloud Drive に配置し、PC/iPhone間で自動同期
- JSONを選択した理由: iCloud同期との相性が良い、テキストエディタで中身を確認できる、年間3,650件程度なのでパフォーマンス問題なし

## 復習HTML

### 日次HTML（毎日生成）

前日分の記録をリッチなHTMLで表示:

- **一文ごとの解析表示**: original → corrected → Why → Key Point
- **折りたたみ**: 各文はクリックで展開/折りたたみ可能（デフォルト展開）
- **Diffハイライト**: Correctionの訂正箇所を緑でハイライト
- **箇条書き**: Whyに複数指摘がある場合はbullet point表示
- **フィルタリング**: 上部のstat card (Corrections/Translations/To Japanese) をクリックでフィルタ
- **カテゴリタグ**: 文法(赤)・表現(オレンジ)・語彙(青)で色分け
- **難易度バッジ**: basic/intermediate/advanced

### 週次HTML（毎週生成）

- 弱点カテゴリの出現回数を棒グラフで表示
- 2回以上繰り返したカテゴリを「繰り返しミス」として表示
- 全記録の一覧

### 生成の仕組み

- macOS launchd で毎朝7時に実行（日次は毎日、週次は毎週月曜）
- JSONデータを読み込み → HTMLテンプレートに流し込み → iCloudフォルダに出力
- ダークテーマ、レスポンシブ対応

## 技術スタック

| レイヤー | 技術 | 理由 |
|---|---|---|
| 入力 | Raycast Extension (TypeScript) | 既存ワークフローに統合 |
| AI (結果) | Google Gemini 3.1 Flash Lite (Raycast AI) | 高速レスポンス |
| AI (解析) | Anthropic Claude 4.6 Sonnet (Raycast AI) | 高品質解析 |
| データ | JSONファイル (iCloud Drive) | シンプル、iCloud同期と相性良好 |
| 復習UI | 静的HTMLファイル (iCloud Drive) | ブラウザで閲覧、PC/iPhone対応 |
| スケジュール | macOS launchd | HTML生成の定期実行 |
